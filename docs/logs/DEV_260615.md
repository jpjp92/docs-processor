# DEV_260615

## Summary

`docs-processor` 루트에 Next.js 기반 PDF 분석 워크스페이스를 구성하고, 기존 `Ref/pdf-workspace`의 핵심 기능을 React/App Router 구조로 옮겼다.

## Project Setup

- Next.js App Router 프로젝트 구조 추가
- React + TypeScript 기반 클라이언트 워크스페이스 구현
- ESLint flat config 구성
- `.gitignore` 추가
- 루트 `README.md` 작성
- `lucide-react` 아이콘 패키지 추가

## Core Features

- PDF 업로드 및 pdf.js 기반 렌더링
- 페이지 썸네일 목록
- 페이지 이동 및 확대/축소
- 확대 후 하단까지 볼 수 있도록 PDF 캔버스 스크롤 레이아웃 보정
- 영역 드래그 선택
- 선택 영역 캡처 이미지 생성
- 선택 영역 내부 텍스트 추출
- 선택 영역별 AI 분석 및 후속 질문

## API

- `GET /api/config`
  - 서버에 설정된 프로바이더 키 목록과 기본 모델 반환
- `POST /api/analyze`
  - 표준 메시지 포맷을 Claude/OpenAI/Gemini 요청으로 변환
  - 서버 키 우선, 없으면 `x-provider-key` BYOK 사용

## Provider Defaults

- Claude: `claude-sonnet-4-6`
- OpenAI: `gpt-4o`
- Gemini: `gemini-2.5-flash`

Gemini 기본 모델은 기존 `gemini-2.0-flash`에서 `gemini-2.5-flash`로 변경했다.

## UI Changes

- 상단바 오른쪽에 설정 아이콘 배치
- 새 문서 분석 아이콘을 설정 아이콘 바로 왼쪽에 배치
- 아이콘 버튼을 기존 UI 톤에 맞게 `paper` 배경과 `line` 테두리로 정리
- 설정 모달 섹션화
  - AI 설정
  - 문서
  - 분석 내보내기
- 설정 모달에서 원본 PDF 다운로드, 다른 파일 열기, JSON 다운로드, HTML 리포트 다운로드 제공

## Export

- JSON 내보내기
  - 문서명, 총 페이지 수, export 시간
  - AI provider/model
  - 선택 영역 이미지
  - 추출 텍스트
  - 질문/답변 turn
  - `finishReason`, `truncated` 메타데이터
- HTML 리포트 내보내기
  - 브라우저에서 바로 열 수 있는 단일 HTML 파일
  - 선택 영역 이미지, 추출 텍스트, 분석 대화 표시
  - 응답 잘림 감지 시 경고 표시

## Truncated Response Handling

실제 테스트 파일:

- `Ref/2024-적극행정-우수사례집-analysis.json`
- `Ref/2024-적극행정-우수사례집-analysis.html`

확인 결과 HTML/JSON export가 응답을 자른 것이 아니라, API 응답 시점의 `assistant.text` 자체가 중간에서 끊긴 상태였다.

조치:

- 서버 `maxTokens`를 `1200`에서 `4096`으로 상향
- Claude `stop_reason`, OpenAI `finish_reason`, Gemini `finishReason` 수집
- 출력 한도 종료 시 `truncated: true` 반환
- UI에서 잘림 경고 표시
- 잘림 감지 시 `이어서 받기` 버튼 표시
- JSON/HTML export에도 잘림 메타데이터 포함

## Development Notes

Next dev 서버가 실행 중인 상태에서 `npm run build`를 실행하면 `.next` 산출물이 충돌해 다음과 같은 오류가 발생할 수 있다.

```text
Cannot find module './873.js'
```

빌드 검증 전에는 dev 서버를 종료하고 `.next`를 정리한다.

```bash
rm -rf .next
npm run build
npm run dev
```
