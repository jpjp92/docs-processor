# DEV_260615

## Summary

`docs-processor` 루트에 Next.js 기반 PDF 분석 워크스페이스를 구성하고, 기존 `Ref/pdf-workspace`의 핵심 기능을 React/App Router 구조로 옮겼다.

## Project Setup

- Next.js App Router 프로젝트 구조 추가
- React + TypeScript 기반 클라이언트 워크스페이스 구현
- ESLint flat config 구성
- `.gitignore` 추가
- 루트 `README.md` 작성
- `lucide-react` 아이콘 패키지 추가

## Core Features

- PDF 업로드 및 pdf.js 기반 렌더링
- 페이지 썸네일 목록
- 페이지 이동 및 확대/축소
- 확대 후 하단까지 볼 수 있도록 PDF 캔버스 스크롤 레이아웃 보정
- 영역 드래그 선택
- 선택 영역 캡처 이미지 생성
- 선택 영역 내부 텍스트 추출
- 선택 영역별 AI 분석 및 후속 질문

## API

- `GET /api/config`
  - 서버에 설정된 프로바이더 키 목록과 기본 모델 반환
- `POST /api/analyze`
  - 표준 메시지 포맷을 OpenAI/Gemini 요청으로 변환
  - Claude는 현재 비활성 처리
  - 서버 키 우선, 없으면 `x-provider-key` BYOK 사용

## Provider Defaults

- Claude: 현재 비활성
- OpenAI: `gpt-5-mini`
- Gemini: `gemini-2.5-flash`

Gemini 기본 모델은 기존 `gemini-2.0-flash`에서 `gemini-2.5-flash`로 변경했다.
OpenAI 기본 모델은 `gpt-5-mini`로 변경했고, Chat Completions 요청에서는 `max_completion_tokens`를 사용하도록 분기했다.

## UI Changes

- 상단바 오른쪽에 설정 아이콘 배치
- 새 문서 분석 아이콘을 설정 아이콘 바로 왼쪽에 배치
- 아이콘 버튼을 기존 UI 톤에 맞게 `paper` 배경과 `line` 테두리로 정리
- 새 문서 분석 아이콘을 `<-` 형태 대신 `FilePlus2` 아이콘으로 교체
- 설정 모달 섹션화
  - AI 설정
  - 문서
  - 분석 내보내기
- 설정 모달에서 원본 PDF 다운로드, JSON 다운로드, HTML 리포트 다운로드 제공
- PDF 편집 기능은 포함하지 않고, 분석/내보내기 중심으로 범위를 유지
- 선택 영역 카드 UI 정리
  - 삭제 버튼을 아이콘화
  - 선택 영역 미리보기와 메타 정보 분리
  - 분석 버튼에 로딩 스피너와 분석 아이콘 적용
  - 질문 전송 버튼을 아이콘화
- AI 응답 렌더링 개선
  - 마크다운 제목, 인용문, 목록, 표, 구분선 스타일링
  - 기본 요약 프롬프트를 구조화해 한줄 요약/핵심 포인트/세부 의미/다음 질문 형식으로 응답 유도
  - 기본 분석 프롬프트는 UI에 사용자 메시지로 노출하지 않음
  - 사용자가 직접 입력한 추가 질문과 프리셋 질문만 `나` 메시지로 표시
- 3분할 레이아웃 조절
  - 왼쪽 페이지 썸네일 영역과 오른쪽 AI 분석 패널 사이에 드래그 핸들 추가
  - 썸네일 영역은 `120px~260px`, AI 분석 패널은 `280px~560px` 범위로 조절
  - 모바일 폭에서는 기존 슬라이드 패널 동작을 유지하고 조절 핸들은 숨김

## Truncated Response Handling

실제 테스트 파일:

- `Ref/2024-적극행정-우수사례집-analysis.json`
- `Ref/2024-적극행정-우수사례집-analysis.html`

확인 결과 HTML/JSON export가 응답을 자른 것이 아니라, API 응답 시점의 `assistant.text` 자체가 중간에서 끊긴 상태였다.

조치:

- 서버 `maxTokens`를 `1200`에서 `4096`으로 상향
- Claude `stop_reason`, OpenAI `finish_reason`, Gemini `finishReason` 수집
- 출력 한도 종료 시 `truncated: true` 반환
- UI에서 잘림 경고 표시
- 잘림 감지 시 `이어서 받기` 버튼 표시
- JSON/HTML export에도 잘림 메타데이터 포함

## Development Notes

Next dev 서버가 실행 중인 상태에서 `npm run build`를 실행하면 `.next` 산출물이 충돌해 다음과 같은 오류가 발생할 수 있다.

```text
Cannot find module './873.js'
```

또는 다음처럼 이전 산출물이 존재하지 않는 vendor chunk를 참조할 수 있다.

```text
ENOENT: no such file or directory, open '.next/server/vendor-chunks/lucide-react.js'
```

빌드 검증 전에는 dev 서버를 종료하고 `.next`를 정리한다.

```bash
rm -rf .next
npm run build
npm run dev
```

로컬 서버 운영 메모:

- `3001` 테스트 서버를 종료하고 필요 시 `3000` 기본 포트로 실행
- 포트 점유 확인:

```bash
ss -tlnp | grep :3000
lsof -i :3000 -P -n
```

- 샌드박스 권한 때문에 `0.0.0.0:3000` 바인딩이 `EPERM`으로 막힐 수 있어, 필요한 경우 외부 권한으로 dev 서버를 실행

## PDF Text Edit Lab

- `docs/PDF_TEXT_OBJECT_EDITING.md`에 PDF content stream 직접 치환 실험을 정리
- `docs/PDF_TEXT_EDIT_INTEGRATION_PLAN.md`에 프로젝트 적용 계획 추가
- `scripts/pdf-selection-replace-lab.mjs`로 선택 영역 기반 치환 랩 구성
- HTML 랩 `tmp/pdf-text-object-lab/selection-replace-lab.html`에서 후보 확인과 수정 PDF 다운로드 테스트 지원
- 성공 조건:
  - text run 감지 가능
  - 기존 폰트의 ToUnicode 역매핑 존재
  - 기존 run과 같은 글리프 수
  - 새 글자가 같은 폰트에 존재
- 미지원 예:
  - `테스트중입 `은 `/F2`에 `테`, `스`, `트`, `중`, `입` 글리프가 없어 direct replacement 실패
- 제품 적용 방향:
  - 초기에는 `PDF 원본 텍스트 치환 실험` 베타 기능으로 분리
  - 서버 API에서 inspect/replace 처리
  - 기존 폰트에 없는 글자는 embedded font overlay fallback으로 분기
- 메인 앱 1차 적용:
  - `lib/pdf-edit/selection-replace.ts` 추가
  - `POST /api/pdf-edit/inspect` 추가
  - `POST /api/pdf-edit/apply` 추가
  - 오른쪽 패널 상단에 `PDF 텍스트 치환 실험` UI 추가
  - 현재 페이지 후보 찾기, 후보 선택, 글리프 수 검사, 수정 PDF 다운로드 연결
- HTTP 테스트:
  - inspect API가 테스트 PDF 1페이지에서 후보 `11`개 반환
  - apply API가 `파악` -> `수요` 치환 PDF 반환
  - 미지원 입력은 `GLYPH_COUNT_MISMATCH`/`MISSING_GLYPH` 계열 에러로 반환
- inspect 보강:
  - 일부 PDF에서 `Could not resolve catalog root.`가 발생해 trailer/xref stream/root 탐색 로직 보강
  - `/Contents [ ... ]` 배열 형태의 페이지 content stream도 순회하도록 수정
  - literal string `TJ`와 `Td` 위치 연산자도 후보로 추출하도록 수정
  - GLM 논문 PDF는 제목 후보 `GLM-5:fromVibeCodingtoAgenticEngineering` 등 `6`개 후보 반환 확인
  - HWP 보고서 PDF는 기존처럼 후보 `11`개 반환 확인
  - trailer `/Root`가 없거나 읽히지 않는 PDF를 위해 `/Type /Catalog` 객체 직접 탐색 fallback 추가
  - page tree의 `/Resources` 상속과 direct resource dictionary 처리 보강
  - `.next` 백업이 `tmp/`에 있을 때 ESLint가 검사하지 않도록 `tmp/**` ignore 추가
- 실제 PDF 추가 검증:
  - `tmp/pdf-text-object-lab/저작권 이슈 트렌드_통권 제28호(2024-5-3호).pdf` 기준 1~5페이지 후보 추출 확인
  - 3페이지에서 후보 `80`개 반환 확인
  - 대표 후보: `있다`, `디퓨전`, `모델은`, `AI`, `뉴스`, `나이트쉐이드`
  - `있다` -> `뉴스` 테스트 치환 PDF 생성 확인
  - 같은 텍스트라도 서로 다른 폰트 레이어가 섞이지 않도록 치환 그룹 조건에 `font`, `fontSize`, `encoding`을 포함
  - `/api/pdf-edit/replace`가 Next dev 서버에서 Server Action 404로 오인되는 현상이 있어 적용 라우트를 `/api/pdf-edit/apply`로 변경
  - 브라우저에서 자동 다운로드가 막히는 경우를 대비해 생성된 PDF object URL을 보관하고 `생성된 PDF 다시 받기` 링크를 표시하도록 보강
  - 글리프 수가 맞아도 embedded subset font에 없는 글자면 실패하므로, 후보별 `supportedChars`를 내려주고 입력 단계에서 미지원 글자를 표시하도록 보강
