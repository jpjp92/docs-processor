# DEV_260701

## Summary

메인 런처, PDF 분석, Refine 이미지 문서화 화면을 제품형 워크플로우에 맞게 정리했다. API 키 입력 UI를 제거하고 서버 `.env` 기반 설정으로 단순화했으며, Word 리포트 품질을 개선했다.

## Launcher Copy & Visual Tone

첫 화면의 흰색 위주 인상을 줄이고 문서 작업 제품처럼 보이도록 런처 레이아웃과 카피를 다듬었다.

- 헤드라인: `읽고, 정리하고, 리포트까지`
- 서브 문구: `PDF는 요약·리포트로, 이미지는 읽어 Word 문서로 정리합니다.`
- 헤드라인은 `리포트까지`가 통째로 다음 줄에 오도록 명시 줄바꿈 처리
- 서브 문구는 데스크톱에서 한 줄로 보이도록 폰트 크기와 줄바꿈 정책 조정
- `/pdf`, `/refine` 진입 카드의 워딩과 아이콘 정리

## Provider & Model Selection

PDF 분석과 Refine 모두 API 키 입력을 UI에서 제거하고, 서버 `.env`에 설정된 키를 기준으로 프로바이더와 모델만 선택하도록 정리했다.

현재 노출 모델:

| Provider | Models |
|----------|--------|
| Gemini | `gemini-3.5-flash`, `gemini-2.5-flash` |
| OpenAI | `gpt-5.4-mini`, `gpt-5.4` |

Claude는 충전 상태 이슈로 계속 비활성화 상태를 유지한다.

## Refine UI & Flow

이미지 문서화 화면의 좌우 레이아웃과 파이프라인 표현을 조정했다.

- 왼쪽 모델 연결 패널은 기본 높이를 오른쪽 시작 화면과 맞춤
- 분석 완료 후에는 왼쪽 패널 높이를 유지하고, 오른쪽 결과 영역만 내부 스크롤로 확장
- `API 키` 입력 영역 제거
- 프로바이더 선택과 모델 선택을 커스텀 드롭다운 스타일로 통일
- 생성 파이프라인 제목 앞에 아이콘 추가
- 파이프라인 우측의 `이미지를 추가하세요` 문구 제거
- 분석 시작 버튼과 업로드 영역 크기 조정

## Refine DOCX Quality

Refine에서 생성되는 Word 문서가 결과 본문 중심으로 보이도록 정리했다.

- 문서 머릿말의 `refine`, `이미지 분석 결과` 류 표현 제거
- 입력 필드의 문서 제목 제거
- 최종 결과에 이미 제목이 포함되므로 docx 생성 단계에서 별도 제목 중복 삽입 제거
- 표 셀과 본문에 남던 `<br>` 태그를 Word 줄바꿈으로 변환
- 인라인 HTML의 `strong`, `em`, 엔티티 문자를 Markdown/텍스트 서식으로 정규화

## Gemini RECITATION Handling

Gemini가 이미지 OCR 과정에서 `RECITATION`으로 빈 응답을 반환하는 경우를 보완했다.

- provider 에러 메시지에 `RECITATION` 원인을 명확히 표시
- Refine OCR 단계에서 원문 재현 대신 관찰·요약 중심의 안전 프롬프트로 1회 재시도
- 재시도 후에도 실패하면 사용자에게 실패 원인을 그대로 보여줌

## PDF Workspace

PDF 분석 페이지의 설정 UI와 내보내기 흐름을 정리했다.

- 설정 모달에서 AI provider/model/API key 항목을 Refine과 같은 톤으로 정리
- `OpenAI API 키가 서버 .env에 연결되어 있습니다` 같은 상태 문구 제거
- JSON 내보내기보다 사용자 친화적인 `Word 리포트`를 기본 다운로드 옵션으로 제공
- HTML 리포트 다운로드는 유지

## PDF DOCX Report Quality

PDF 분석 결과 Word 리포트가 원시 Markdown 덤프처럼 보이는 문제를 개선했다.

- 모델 답변의 Markdown을 제목, 인용, 목록, 번호 목록, 표, 인라인 강조 서식으로 변환
- `## 한줄 요약`, `>`, `- **...**` 같은 문자가 그대로 보이지 않도록 처리
- 첫 제목은 `PDF 분석 리포트`로 고정하고 파일명은 메타 테이블에 표시
- 선택 영역별 `추출 텍스트`, `질문`, `답변`, `상태` 라벨을 문서 섹션처럼 정리
