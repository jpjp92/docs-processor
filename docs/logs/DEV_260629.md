# DEV_260629

## Summary

루트 진입 화면을 작업 선택 런처로 바꾸고, 기존 PDF 분석 워크스페이스와 이미지 기반 Word 문서 생성기(Refine)를 별도 라우트로 분리했다.

- `/`: 작업 선택 런처
- `/pdf`: 기존 PDF 분석 워크스페이스
- `/refine`: 이미지 → OCR/통합 검토/해석/docx 생성 워크스페이스
- PDF 분석 화면과 Refine 화면 간 상호 이동 버튼 추가
- `docx`, `marked` 의존성 추가

## Launcher & Routes

기존 `app/page.tsx`가 바로 `PdfWorkspace`를 렌더링하던 구조에서, 첫 화면이 작업을 고르는 런처가 되도록 변경했다.

| Route | 역할 |
|-------|------|
| `/` | PDF 분석 / 이미지 문서화 작업 선택 |
| `/pdf` | 기존 PDF 열람, 영역 선택, AI 분석, 요약 |
| `/refine` | 이미지 기반 docx 생성 |

PDF 워크스페이스 상단에는 `/refine` 이동 링크를 추가했고, Refine 상단에는 `/`와 `/pdf` 이동 링크를 추가했다.

## Refine Workflow

`Ref/refine-react` PoC를 Next 앱 구조에 맞춰 이식했다. Vite 프록시 대신 기존 `/api/analyze` Route Handler를 재사용한다.

주요 흐름:

1. 이미지 업로드 및 미리보기
2. 이미지별 병렬 OCR/구조 추출
3. 인쇄 쪽번호 감지 및 정렬
4. 추출 Markdown 조각 통합 검토
5. 문서형 구조 보강
6. Markdown 렌더링 및 `.docx` 생성

추가 모듈:

| 모듈 | 내용 |
|------|------|
| `components/refine-workspace.tsx` | Refine UI와 파이프라인 오케스트레이션 |
| `lib/refine/prompts.ts` | OCR, 통합 검토, 문서 보강 프롬프트 |
| `lib/refine/parser.ts` | `@@PAGE`, `@@NOTES`, `@@DOCUMENT` 파서 |
| `lib/refine/pool.ts` | 동시 실행 제한 병렬 풀 |
| `lib/refine/md2docx.ts` | Markdown → Word 문서 변환 |

## Document Quality

초기 결과가 표 추출 중심으로만 나오거나 `@@NOTES` 마커가 본문에 섞이는 문제가 있어 후처리를 강화했다.

- `@@NOTES`, `@@DOCUMENT` 제어 마커 제거
- 최종 문서에서 `이미지 1`, 파일명, 업로드 순번을 제목으로 쓰지 않도록 프롬프트 강화
- `# 내용 기반 제목` → `## 핵심 요약` → 표 → `### 읽는 법/해석` 구조 유도
- 핵심 요약/표 해석이 빠진 경우 문서 보강 패스를 한 번 더 실행
- 최종 문서 끝에 `## 종합 평가` 섹션을 강제하고, 핵심 요약/해석/종합 평가 항목을 Markdown 불릿으로 정리하도록 보강
- docx 변환 단계에서 핵심 요약/읽는 법/해석 섹션의 짧은 나열 문장을 실제 불릿 목록으로 보정
- docx 스타일 개선: 표지성 상단 블록, 헤더/푸터, 제목 계층, 표 헤더 음영, 셀 여백, 문단 간격

## Token Limits

이미지 2장 이상에서 표와 해석이 길어질 때 통합 검토 결과가 중간에 끊기는 문제가 있었다.

- `/api/analyze`가 요청별 `maxTokens`를 받을 수 있도록 확장
- 서버 허용 상한: `32,000`
- Refine 단계별 요청 한도:
  - 이미지별 OCR: `4,096`
  - 통합 검토: `24,000`
  - 문서 보강: `28,000`
- provider가 `truncated`를 반환하면 불완전한 docx 생성을 중단하고 오류로 알림

## UI Updates

- Refine 파이프라인을 `병렬 추출 → 통합 검토 → docx 생성` 흐름으로 시각화
- 진행 중인 단계에 회전 링, 펄스, sweep 애니메이션 추가
- `prefers-reduced-motion` 환경에서 애니메이션 비활성화
- `검토본·해석`, `추출본`, `변경 사항` 탭 구성
- 표 렌더링 시 첫 열이 지나치게 좁아지지 않도록 화면 CSS 개선
- `새 분석` 버튼 추가: 이미지, 결과, docx blob, 파이프라인 상태를 한 번에 초기화

## API Notes

`.env` 연계는 기존처럼 서버에서 처리한다.

- Gemini: `GEMINI_API_KEY` 우선, 없으면 `GOOGLE_API_KEY`
- OpenAI: `OPENAI_API_KEY`
- Claude: `ANTHROPIC_API_KEY`는 읽지만 현재 비활성화

Gemini 빈 응답/차단/`finishReason` 정보를 더 구체적으로 화면까지 전달하도록 provider 에러 처리를 개선했다.
