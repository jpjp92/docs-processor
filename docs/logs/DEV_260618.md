## Whole-document Summary (Phase 1)

### 산출물 스펙

```text
개요  : 문서 전체를 한 문장(한 줄)
섹션  : [섹션명] — 한 줄  × 최대 10개
```

- 모든 항목은 한 줄을 넘기지 않는다(개행 제거 + 길이 컷).
- 섹션은 모델이 본문의 논리적 흐름(서론/배경/쟁점/사례/결론 등)을 추론해 구성한다.
- 한 줄/10섹션 제한은 프롬프트에만 의존하지 않고 파싱 단계에서 강제한다.

### Library Layer

- [lib/summarize.ts](../../lib/summarize.ts)
  - `SUMMARY_INSTRUCTION`: 라인 기반 출력 형식 프롬프트
    - `OVERVIEW: <한 줄>`
    - `SECTION: <섹션명> || <한 줄>`
  - `buildSummaryMessages(text)`: provider 표준 메시지로 변환
  - `parseSummary(raw)`: 라인 파싱 → `{ overview, sections }`
    - 스트리밍 중 부분 텍스트로 호출돼도 안전(점진 렌더용)
    - `MAX_SECTIONS = 10`, 줄당 약 90자 상한 강제
  - `MAX_INPUT_CHARS = 600_000`: 단일 호출 안전 상한(초과 시 Phase 2 분기)

### Provider Streaming

- [lib/providers.ts](../../lib/providers.ts)에 SSE 스트리밍 호출 추가
  - `streamOpenAI`: `stream: true` + `data:` 델타 파싱
  - `streamGemini`: `streamGenerateContent?alt=sse` + parts 텍스트 파싱
  - `iterateSse`: 공통 SSE 라인 파서
  - `streamProvider(provider, options)`: 어댑터 진입점(claude는 미지원)
- 기존 비스트리밍 `callProvider`는 그대로 유지(영역 분석용).

### API Layer

- `POST /api/summarize` ([app/api/summarize/route.ts](../../app/api/summarize/route.ts))
  - 입력: `{ text, provider?, model? }`
  - 기본 provider는 **Gemini Flash**(롱컨텍스트 + 저비용)
  - `MAX_INPUT_CHARS` 초과 시 `413` + `reason: "INPUT_TOO_LONG"`
  - 본문 누락 시 `400`, 키 누락 시 `401`
  - 출력: `text/plain` `ReadableStream`(모델 텍스트 그대로 스트리밍)

### UI

- [components/pdf-workspace.tsx](../../components/pdf-workspace.tsx)
  - `collectAllText(doc)`: 전 페이지 `getTextContent`를 모아 본문 구성
  - `runSummary()`: 스트림을 읽으며 `parseSummary`로 개요/섹션을 실시간 갱신
  - 우측 패널 상단에 `문서 전체 요약` 섹션 + `요약 생성` 버튼
  - 문서 로드/초기화 시 요약 상태 리셋
  - 스캔 PDF 등 본문 텍스트가 없으면 안내 메시지 표시
- [app/globals.css](../../app/globals.css)
  - 개요: 왼쪽 액센트 바 + 본문 강조
  - 섹션: 상단 구분선으로 분리, 여백 중심의 깔끔한 행 구성

### Export 포함

- `buildAnalysisExport()` 페이로드에 `summary: { overview, sections } | null` 추가
- HTML 리포트: hero 바로 아래(셀렉션 목록보다 위) **최상단**에 `Document Summary` 블록 렌더
- JSON export에도 동일하게 요약 포함
